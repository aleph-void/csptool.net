Before You File an Issue

A few quick checks that help us help you faster.

1

Search Existing Issues

Someone may have already reported it. A thumbs-up on an existing issue helps us prioritize.

2

Include Details

Browser, extension version, the URL or CSP involved, and clear repro steps make a huge difference.

3

Submit

File the issue on GitHub. We'll triage and respond as soon as we can.
